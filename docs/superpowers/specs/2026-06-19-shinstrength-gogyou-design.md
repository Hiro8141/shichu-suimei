## 0. 背景

既存アプリは命式＋通変星＋十二運＋基本性格/才能までを表示する。本機能は既存の `meishiki`（日主・各柱の干支・蔵干・通変星・五行カウントを保持）を土台に、**日主の強弱（身強・中庸・身弱）判定**と**五行バランスのレーダー可視化**を追加し、結果画面の「五行の傾向」カードをリッチな分析セクションに拡張する。

後続機能（②大運・年運、③格局・用神、④相性診断）はそれぞれ別の設計書・実装サイクルで進める。本書は①のみを対象とする。

## 1. スコープ

含む：
- 身強・中庸・身弱の判定エンジン（簡易スコア法・月令重視）
- 五行レーダー（五角形SVG）の生成
- 判定結果の開運アドバイス文／五行の過不足コメント（本プロジェクトで執筆）
- 結果画面への統合（既存「五行の傾向」カードを置き換え）

含まない（後続機能）：
- 大運・年運、格局・用神、相性診断
- 五行カウント方式の刷新（既存の「天干＋地支本気」カウントをそのまま使う）

## 2. 身強身弱エンジン `js/engine/shinStrength.mjs`

純粋関数。入力は既存 `meishiki`、出力は判定オブジェクト。`tenStar`（既存）を再利用して各干支を日主から見た五大分類に振り分ける。

### 2.1 分類

日主（日干）から見た関係で、各干支を「味方／敵」に分類する。

- **味方（support, +）**：比劫（比肩・劫財＝日主と同じ五行）、印（印綬・偏印＝日主を生む五行）
- **敵（drain, −）**：食傷（食神・傷官）、財（正財・偏財）、官殺（正官・偏官）

判定には既存 `tenStar(dayStem, otherStem)` を使う。地支は **本気の蔵干**（`hiddenStems[0]`）に対して `tenStar` を適用して分類する。

### 2.2 スコアリング（簡易法・月令重視）

対象とする文字：
- 天干：年・月・（時）の天干。**日干（=日主そのもの）は除外**
- 地支：年・月・日・（時）の地支（本気）

各文字は味方なら `+`、敵なら `−` の符号付きで加点する。

- 基本の重み：各文字 ±1
- **月支（月令）は最重要のため重み ±3**（得令なら大きく加点、失令なら大きく減点）
- 時柱がない場合は時の天干・地支を対象から外す（3柱で集計）

合計を `score` とする。

### 2.3 判定の帯

- `score >= +2` → **身強**
- `score <= -2` → **身弱**
- それ以外（拮抗）→ **中庸**

閾値（±2）と月令の重み（±3）はテストで既知サンプルに合わせて調整する。設計上の初期値であり、実装時に身強寄り・身弱寄り・拮抗の検証サンプルで微調整してよい。

### 2.4 出力

```
{
  level: '身強' | '中庸' | '身弱',
  score: number,        // 合計スコア（符号付き）
  support: number,      // 味方側の加点の絶対量（透明性のため）
  drain: number,        // 敵側の減点の絶対量
}
```

## 3. 五行レーダー `js/app/radarChart.mjs`

純粋関数：五行カウント `{wood,fire,earth,metal,water}` を受け取り、五角形レーダーの **SVG文字列** を返す。DOMへの挿入は呼び出し側（`main.mjs`）が行う。

- 頂点配置：木＝上、時計回りに 火・土・金・水
- 各五行カラー：木=#3ad6a0（青緑）／火=#ff6b6b（赤）／土=#e7c14a（黄）／金=#cfd6e6（銀）／水=#5aa0ff（青）
- 背景の基準五角形（最大値の目安）＋データ五角形を重ねる。各頂点に小さな星（点）と五行ラベル
- カウントの最大値で正規化（全0なら中心に縮約しても破綻しないこと）
- `viewBox` 指定＋ `width:100%` で iPhone SE でも潰れない可変幅

## 4. 診断文の追加 `js/data/interpretations.mjs` / `interpret.mjs`

`interpretations.mjs` に追記：
- `STRENGTH`：`身強/中庸/身弱` ごとの **傾向＋開運アドバイス文**（本プロジェクトで執筆）
  - 例（身強）：「自分の力が満ちているタイプ。エネルギーを外へ出す“与える・動く”生き方が吉。抱え込むより発散を」
  - 例（身弱）：「周囲と支え合って力を発揮するタイプ。無理に背伸びせず、人や環境の助けを借りると安定します」
  - 例（中庸）：「力のバランスが取れたタイプ。状況に応じて攻守を切り替えられる柔軟さが強み」
- 五行の **過不足コメント**：不足（カウント0）の五行があれば「〇が不足気味。〇を補うと安定します」の一文を持つ短文テーブル

`interpret.mjs` の戻り値に追加：
- `strength`：`{ level, advice }`（level は判定、advice は上記アドバイス文）
- `lackingElements`：カウント0の五行キー配列とそのコメント

既存の `dayMaster` / `talents` / `dominantElement` は維持する。

## 5. UI統合とレスポンシブ

- `viewModel.mjs`：`buildViewModel` の戻り値に以下を追加
  - `radarCounts`：`meishiki.fiveElements`（レーダー入力用）
  - `strength`：`{ level, advice }`
  - `lacking`：不足五行コメント（あれば）
- `main.mjs` の `renderResult`：既存の「五行の傾向」カードを、以下を含む新セクションに差し替え
  - 五行レーダー（`radarChart` のSVGを挿入）
  - 身強/中庸/身弱の判定バッジ
  - 開運アドバイス文＋（あれば）五行の過不足コメント
- レスポンシブ：レーダーSVGは可変幅。iPhone SE（375px）でセクションが崩れないこと

## 6. テスト

- `shinStrength`（`tests/shinStrength.test.mjs`）
  - 味方が多く月支が得令 → 身強
  - 敵が多く月支が失令 → 身弱
  - 拮抗 → 中庸
  - 月支の得令／失令で判定が変わることを確認
  - 時柱なし（3柱）でも集計が成立すること
- `radarChart`（`tests/radarChart.test.mjs`）
  - 戻り値SVGに5頂点・5つの五行ラベルが含まれる
  - 全カウント0でも例外なくSVGを返す
- `interpret`（既存テストに追記）
  - `strength.level` と `strength.advice` が組み立つ
  - 不足五行があるとき `lackingElements` が出る
- 既存32テストが引き続き全て通ること
